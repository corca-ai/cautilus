## Findings

### Act Before Ship

- Schema-validation harness weakness is now part of the blocker.
  The next implementation must not only harden `fixtures/robustness/*.schema.json`; it must prove those hardening rules fail bad packets.
  Do not build a full JSON Schema engine.
  Extend only the subset needed by the robustness schemas: `additionalProperties: false`, `oneOf`, `if` / `then`, and `not` with `required`.
  Add targeted negative tests for each new schema rule, and fail robustness schema tests if unsupported hardening keywords are introduced silently.
- Provenance/ref shape needs a precise common contract.
  `schemaVersion` is metadata, not a locator.
  Ref-bearing fields should require `kind` plus at least one real locator from `path`, `ref`, `url`, or `artifactId`.
  Apply that shape to `requestRef`, `planRef`, `subjectRef`, `baselineRef`, `variantRef`, `sourceEvidenceRefs[]`, `artifactRefs[]`, and `targetRefs[]` where present.
- Mutation-kind conditional hardening needs both required fields and exclusion.
  In `cases[]`, `mutationKind: stimulus` should require `stimulusKind` and forbid `implementationKind`.
  `mutationKind: implementation` should require `implementationKind` and forbid `stimulusKind`.
  Do not extend this rule to report `caseResults[]` until report results carry those detail fields.
- Required robustness arrays must be non-empty.
  Add `minItems: 1` to request `riskFocus`, `sourceEvidenceRefs`, and `requestedMutationKinds`; plan `cases` and `cases[].sourceEvidenceRefs`; report `caseResults` and `nextActions`.
  Do not apply this to optional diagnostic arrays such as `limitations` or `reasonCodes`.

### Bundle Anyway

- Update examples alongside schema hardening so they use the same common ref shape.
- Add the targeted schema checker sentinel while editing the checker, so unsupported hardening keywords do not become false confidence.

### Valid But Defer

- Add deterministic critique-packet coverage for robustness schema rules later if this surface becomes a repeated review target.
  The current packet only covers `release-packaging` and `cli-agent-product` rule families, and it says broader critique still applies.

### Over-Worry

- A full draft-2020-12 JSON Schema validator or repo-wide schema infrastructure rewrite is unnecessary for this slice.
  The gate should cover the exact schema keywords the robustness packet contract uses.
