## Findings

### Act Before Ship

- Required robustness arrays can be empty.
  `riskFocus`, `sourceEvidenceRefs`, `requestedMutationKinds`, `cases`, `caseResults`, and `nextActions` are required fields but currently lack `minItems`.
  Empty arrays would let a request ask for no mutation kind, a plan contain no explicit cases, or a report contain no results.
- Provenance refs are too weak.
  `requestRef`, `planRef`, `baselineRef`, `variantRef`, `sourceEvidenceRefs[]`, and `artifactRefs[]` are mostly bare objects, so `{}` can validate despite the contract saying Cautilus owns auditable provenance fields.

### Bundle Anyway

- Mutation-kind-specific detail should be tightened cheaply.
  If `mutationKind` is `stimulus`, require `stimulusKind`; if it is `implementation`, require `implementationKind`.
  Do not force `baselineRef`, `variantRef`, `artifactRefs`, or per-case summary yet.
- The Go mini schema checker ignores `additionalProperties: false`.
  Adding that check plus a small negative test would improve fixture confidence while touching the same validator.
- The example report uses `/tmp/...` artifact refs.
  Replace them with repo-relative illustrative refs or explicitly mark them as non-durable runtime outputs.

### Valid But Defer

- Cross-field report semantics should become a dedicated validator later.
  JSON Schema can shape `summary`, but `summary.totalCases == len(caseResults)` and `expectedRelation` / `observedRelation` / `relationStatus` consistency belong in a runtime or fixture validator slice.

### Over-Worry

- Broad CLI/runtime robustness execution is still out of scope.
  The critique did not find drift into Charness quality ownership, improve-search mutation ownership, or an implied runtime flag.

## Next Move

Patch the schemas before treating commit `517ad37` as final for issue 44:

- Add `minItems: 1` to the core request, plan, and report arrays.
- Replace bare provenance object shapes with a small shared ref shape requiring `kind` plus at least one stable locator such as `path`, `ref`, or `schemaVersion`.
- Bundle the mutation-kind conditional requirements while editing `plan.schema.json`.
